    分别封装起来，让他们之间可以相互替换，此模式让算法的变化独立于使用算法的客户

## 2、观察者模式

## 3、装饰对象
    开放-关闭原则

    1、开放：欢迎使用任何想要的行为来扩展类

    2、关闭：我们花了很多时间得到了正确的代码，还解决了所有一直的bug。所以你不能再修改现有的代码。
        我们必须关闭代码防止被修改。
    
    我们的目标是允许类的扩展。在不修改再有代码的情况下，就可以搭配新的行为。
